# 工作目标：自然语言 → CUDA 前反向 Kernel 自动生成 Agent

## 一、任务定义

用户用**自然语言**描述一个类 attention 的算法结构（附带张量的 shape 与 dtype 约定），agent 自动完成如下闭环：

1. **PyTorch 参考实现**：由自然语言描述生成一份可运行的 PyTorch 参考实现（含 autograd 前向反向），作为正确性金标准。
2. **CUDA Kernel 生成**：生成自定义 CUDA 前向 + 反向 kernel。
3. **优化迭代**：基于 profile 反馈进行算子融合、访存合并、shared-memory tiling、float4 向量化、warp 原语、TF32 等优化。
4. **交付**：产出可独立编译的 `.cu` 文件。

**边界约束**：允许在 kernel 内调用 cuBLAS/cuDNN；交付/待测路径上**禁止**落回 `F.scaled_dot_product_attention` / `torch.nn.functional` 等高层算子。

## 二、输入 / 输出契约

- **输入**：类 attention 结构的自然语言描述 + 张量 shape/dtype 约定。
- **中间产物**：PyTorch 参考实现（正确性金标准）。
- **输出**：可独立编译的 `.cu` 文件（含必要 host 绑定），前反向 kernel 接口清晰、无对 PyTorch 高层算子的运行时依赖。

## 三、正确性判据（前向 + 反向都要过）

以 PyTorch 参考实现为金标准：

- 用 **≥5 组独立随机输入**（不同随机种子）。
- 前向输出：`allclose(atol=1e-2, rtol=1e-2)`。
- 反向梯度：对所有相关张量的梯度同样 `allclose(atol=1e-2, rtol=1e-2)`。
- 小规模用例用**双精度 `torch.autograd.gradcheck`** 复核反向。
- 任一不通过即判该用例失败。

## 四、完成度检验（核心验收条件）

选定**一个现成的、具体的算法结构**作为验收用例（例如标准 causal multi-head attention，固定 batch / head 数 / seq_len / head_dim / dtype）。在该用例上：

- 正确性通过（前向 + 反向），**且**
- 自定义 CUDA kernel 相对 `torch.compile`（默认 mode）**前向与反向均快 ≥5%（≥1.05×）**

即视为达成目标。

## 五、计时协议（防测量偏差 / 防作弊）

- 同一 GPU、同一输入；单块本地 NVIDIA GPU。
- warmup ≥ 10 次；正式测量 ≥ 100 次取几何均值。
- 每次计时前后 `torch.cuda.synchronize()`，用 CUDA events 计时。
- 3 次重跑控制方差，CV > 5% 作废重测。
- **前向、反向分别计时并各自达标**。
- 评测 / 计时脚本对 agent 只读且进程隔离，禁止改动。
- 禁止降精度换速度（除非描述本身指定低精度）。

## 六、防作弊清单

- 禁止在待测路径落回 `torch.nn.functional` / SDPA 等高层算子。
- 禁止修改 / 绕过 verify 与计时脚本。
- 禁止降精度换速度（原描述指定低精度除外）。
- 交付的 `.cu` 须能独立编译、无 torch 高层运行时依赖。

## 七、一句话总结

用户用自然语言描述一个类 attention 结构，agent 自动完成 "torch 参考实现 → CUDA 前反向 kernel → 优化 → .cu 交付" 的闭环；在一个选定的现成算法结构（如标准 causal MHA）上，以 torch 参考为金标准通过前反向正确性（≥5 组随机输入、atol=rtol=1e-2），并在规范计时下让前向和反向都比 torch.compile 快至少 5%，即达成目标。
