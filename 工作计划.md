## 阶段 0 — 验收基础设施（GPU 无关，可先写；在 Colab 验证）

> 这套设施双重身份：既是**判定 skill 是否达标的评测器**，也是 **skill 交给宿主 agent 的只读工具**。按防作弊要求，计时/验证脚本对 agent 只读、进程隔离。

- [ ] **0.1** 项目骨架：目录结构（`skill/` `reference/` `tests/` `benchmarks/` `kernels/` `notebooks/` `scripts/`）、`README.md`、`requirements.txt`。
- [ ] **0.2** 敲定验收用例精确规格：`reference/config.py`，固定 causal MHA 的 batch、heads、seq_len、head_dim、dtype、随机种子列表（≥5）。
- [ ] **0.3** 手写 PyTorch 参考实现 `reference/mha_reference.py`：causal MHA 前向 + autograd 反向，作为正确性金标准（基础 matmul+softmax+mask 表达，不落回 SDPA）。
- [ ] **0.4** 正确性验证器 `tests/verify.py`：传入一个 kernel 实现即可复用；≥5 组随机种子跑前向 allclose(atol=rtol=1e-2) + 反向各梯度 allclose；小规模 double `gradcheck`。任一不过即判失败。
- [ ] **0.5** 计时基准器 `benchmarks/bench.py`：warmup≥10、正式≥100 取几何均值、CUDA events、每次前后 synchronize、3 次重跑算 CV、CV>5% 作废重测；前反向分别计时；对比 `torch.compile`(默认 mode)。**设计为只读、进程隔离、不被 agent 改动**。
- [ ] **0.6** Colab 驱动 notebook `notebooks/run.ipynb`：git clone/pull + 装依赖 + 运行探测/验证/基准入口。先跑通"环境探测"确认回环可用。

## 阶段 1 — 手写参考 kernel 打通管线（自检 + 作为 skill 的参考解/调试样本，非最终交付）

> 目的：证明"编译→正确→计时→对比 torch.compile"这条评测管线本身可用，并摸清 T4 上打赢 torch.compile 的难度；产物同时作为 skill 提示里的正确解示例。

- [ ] **1.1** CUDA 扩展加载脚手架 `kernels/loader.py`：`torch.utils.cpp_extension.load` 即时编译，参数针对 sm_75。
- [ ] **1.2** 手写 causal MHA **前向** kernel（朴素正确版）`kernels/mha_forward.cu`。
- [ ] **1.3** 前向接入验证器：allclose 对拍参考实现通过（≥5 组种子）。
- [ ] **1.4** 手写 causal MHA **反向** kernel（朴素正确版）`kernels/mha_backward.cu`：dQ/dK/dV。
- [ ] **1.5** 反向接入验证器：各梯度 allclose + 小规模 gradcheck 通过。
- [ ] **1.6** 首次基准：手写 kernel vs torch.compile 前反向分别计时，记录 baseline（此时大概率打不过，用于摸差距）。

## 阶段 2 — 编写 Skill（方法论指令 + 工具，宿主无关）

- [ ] **2.1** Skill 设计文档 `skill/DESIGN.md`：skill 的输入/输出、agent 每步该做什么、可用工具（验证器/基准器/加载器）、失败反馈如何回喂、防作弊红线。
- [ ] **2.2** 编写 `skill/SKILL.md`（方法论主体）：指导宿主 agent 从 NL 描述 → PyTorch 参考 → CUDA 前反向 → 用验证器自检 → 用基准器测速的完整步骤与判据（自然语言，宿主无关）。
- [ ] **2.3** Skill 工具封装 `skill/scripts/`：把验证器、基准器、编译加载封装成命令行工具（agent 一条命令可调用，只读评测脚本），带清晰 stdout 反馈供 agent 读取。
- [ ] **2.4** loop 驱动脚本 `skill/loop.md`（或 py）：定义"生成→编译→验证→读 profile→改进"的迭代循环与终止条件（正确且前反向均达标 / 到达轮次上限）。
- [ ] **2.5** Skill 干跑测试：喂 causal MHA 的 NL 描述，宿主 agent 跑到"正确性通过的前反向 kernel"（先不要求达速度标）。

## 阶段 3 — 优化冲刺（由 skill/loop 驱动宿主 agent，打赢 torch.compile ≥5%）

- [ ] **3.1** profile 反馈接入 skill：让 agent 能读 profiler 输出（访存/占用率/bank conflict）并据此改 kernel。
- [ ] **3.2** 在 skill 指导下迭代**前向**优化：tiling + float4 + warp 原语 + 融合，逼近并超过 torch.compile 前向。
- [ ] **3.3** 在 skill 指导下迭代**反向**优化：同类手段超过 torch.compile 反向。
- [ ] **3.4** 达标确认：规范计时下前反向均 ≥1.05×，3 次重跑 CV≤5%，正确性仍通过。（如 T4 难达标，评估升级 GPU。）

## 阶段 4 — 端到端验收与交付

- [ ] **4.1** Skill 端到端验收：以 Claude Code 为宿主，仅凭 NL 描述 + skill，自动产出通过正确性且前反向均 ≥1.05× 的 `.cu`。
- [ ] **4.2** 交付自检：`.cu` 独立编译通过 + 防作弊清单逐条核对（无 SDPA/F.* 落回、未改计时脚本、未降精度、无 torch 高层运行时依赖）。
- [ ] **4.3** Skill 通用性说明 + README：记录如何挂到其他宿主（Codex/开源 agent），收尾。
